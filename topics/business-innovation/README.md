# 🎯 高雄捷運數位轉型創新提案 - IBM Bob Workshop

> **目標對象**：經理、副理、產品負責人、業務分析師（無需技術背景）
> **難度**：⭐ 入門（完全不需要寫程式）
> **時長**：60 分鐘 + 5 分鐘 Pitch
> **分工模式**：4 人團隊協作（創意發想 + 提案製作）

## 🎪 Workshop 情境

你們是高雄捷運公司的**數位轉型專案小組**，公司已經有三個系統正在運作：
- 💳 **信用卡交易監控系統**（聯邦網通合作）
- 🚇 **捷運站務管理系統**
- 🗺️ **旅程分析系統**

**任務**：使用 IBM Bob 分析這三個系統，提出一個**創新的數位服務提案**，在 60 分鐘內完成提案簡報，並進行 5 分鐘 Pitch！

## 👥 4 人分工流程（60 分鐘）

### Phase 1：啟動與選題（0-10 分鐘）

#### 全員討論
1. **選擇提案主題**（3 分鐘）
   - 快速投票決定 A/B/C/D
   - 確認創新方向

2. **分配角色任務**（4 分鐘）
   - Team Leader：定義目標與 KPI
   - Business Analyst：分析系統與商機
   - Innovation Designer：設計服務流程
   - Presentation Master：規劃簡報結構

3. **快速 Kick-off**（3 分鐘）
   - 確認產出物
   - 同步時間點

---

### Phase 2：分析與發想（10-35 分鐘）

#### 👔 Team Leader 任務（25 分鐘）
**使用 Bob 定義專案目標**

```
我是高雄捷運的數位轉型專案經理，我們要推出「[選定的提案主題]」

請幫我：
1. 定義這個專案的商業目標（3-5 個）
2. 設定可衡量的 KPI 指標
3. 分析目標客群（至少 3 種人物誌）
4. 評估潛在風險與挑戰
5. 規劃 6 個月的執行時程

請以專業的專案企劃書格式呈現
```

**產出**：專案目標與執行計畫（精簡版 Markdown）

---

#### 💼 Business Analyst 任務（25 分鐘）
**使用 Bob 分析現有系統**

**Step 1：分析三個系統**
```
請分析以下三個系統的功能與資料：
1. topics/transaction-monitor/ - 信用卡交易監控
2. topics/station-system/ - 捷運站務管理
3. topics/journey-analysis/ - 旅程分析

針對「[選定的提案主題]」，說明：
- 每個系統可以提供什麼資料？
- 這些資料如何支持新服務？
- 有哪些商業洞察？
```

**Step 2：商業價值分析**
```
基於剛才的分析，產生商業價值報告：
1. 市場機會分析
2. 競爭優勢
3. 收益模式（至少 3 種）
4. 成本效益評估
5. 投資回報預估（ROI）

請用商業簡報的語言，避免技術術語
```

**產出**：商業價值分析報告（精簡版 Markdown）

#### 🎨 Innovation Designer 任務（25 分鐘）
**使用 Bob 設計服務體驗**

**Step 1：使用者旅程設計**
```
我要設計「[選定的提案主題]」服務

請幫我設計完整的使用者旅程：
1. 使用者如何發現這個服務？

3. 日常使用情境（3-5 個場景）
4. 獎勵/回饋機制

每個步驟都要說明使用者的想法和感受

**Step 2：服務藍圖**
```
產生服務藍圖（Service Blueprint）：
1. 前台：使用者看到的介面
2. 後台：系統需要做的事
3. 支援流程：需要哪些資料和功能
4. 關鍵接觸點：使用者與系統互動的時刻

以表格或流程圖呈現
```

**產出**：使用者旅程與服務藍圖（精簡版 Markdown）

3. **使用 Bob 優化**（4 分鐘）
   ```

### Phase 4：排練與 Pitch（55-65 分鐘）

## 💡 Bob 使用技巧（非技術人員專用）

### 1. 如何問出好問題？

**❌ 不好的問題**
```
幫我做一個提案
```

**✅ 好的問題**
```
我要做「捷運集點遊戲化」提案，目標是提升搭乘率 20%
請幫我分析：
1. 目標客群是誰？
2. 他們的需求是什麼？
3. 競爭對手有哪些？
4. 我們的優勢在哪？
```

### 2. 如何讓 Bob 產生更好的內容？

**提供背景資訊**
```
我是高雄捷運的專案經理，我們有 50 個站點，日運量 20 萬人次
現在要推出...
```

**明確指定格式**
```
請以表格呈現
請用商業語言，不要技術術語
請加入具體數字和案例
```

### 3. 如何優化 Bob 的回答？

**分段處理**
```
第一次：請先分析市場機會
第二次：根據剛才的分析，設計收益模式
第三次：評估投資回報
```

**要求改善**
```
這個提案太保守，請加入更創新的想法
這個數字不夠有說服力，請提供更具體的預估
```

## 🆘 常見問題

### Q1: 我完全不懂技術，可以參加嗎？
**A**: 可以！這個 Workshop 就是為非技術人員設計的。你只需要：
- 會使用 Bob 提問
- 會閱讀和理解 Bob 的回答
- 會複製貼上內容
- 有創意和商業思維

### Q2: 如果 Bob 給的答案不符合需求？
**A**: 
1. 更明確地描述你的需求
2. 提供更多背景資訊
3. 要求 Bob 重新思考：「請用更創新的角度重新分析」
4. 請團隊成員協助

### Q3: 90 分鐘會不會太趕？
**A**: 時間分配建議：
- 前 15 分鐘：選題與分工
- 中間 30 分鐘：各自產出（平行作業）
- 後 25 分鐘：整合與優化
- 最後 20 分鐘：排練與準備

如果時間不夠，可以簡化某些部分，重點是完成一個完整的提案。

### Q4: 如何讓提案更有說服力？
**A**: 
- 用數據說話（市場規模、成長率、ROI）
- 說故事（使用者情境、痛點、解決方案）
- 視覺化（圖表、流程圖、示意圖）
- 對比（現況 vs 未來、我們 vs 競爭對手）
